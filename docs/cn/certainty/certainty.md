# 确定性


## AI & 软件
* 软件还是软件, 需要解决真实的问题
* AI大大提高了开发效率
  * AI在复杂度不高的场景、提效不至10倍
  * AI生成的单元测试在80分以上
* 当前(2026年)AI生成的软件质量一般。
* AI短期会提高团队的管理难度，长期可能降低团队的管理难度。 
* 


## 软件
* 软件质量的诉求与软件的生命周期有关。
  * 长生命周期的软件，对质量的诉求，高于对开发效率的追求。
  * 高质量的软件很少见，无论是开源还是商业软件，AI没有改变这一局面。
* 软件最大的敌人是软件复杂度。
  * 控制规模可以降低复杂度。
  * 依赖成熟的软件可以降低复杂度。
* 基于云的软件生态会继续发展。
  * 更丰富的PAAS生态
  * 更丰富的SAAS生态

## 软件工程师
* AI时代对软件工程的能力要求变高了。
  * 持续读书与写作，是AI时代，软件工程师的基本功。
  * AI加速了软件岗位的融合(前+后、开发+测试、产品+开发、...)
  * 简单的问题会被AI解决，少量复杂的问题留给人。
* 软件工程师为软件负责，无论手写还是Vibe
* 软件设计依然是软件工程师的重要能力
* Code Review依然是软件工程师的重要能力
* 软件是软件工程师的立身之本
  * 参与顶尖的软件研发，是极具价值的职业规划
